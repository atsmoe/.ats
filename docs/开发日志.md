## 4. 模板架构

- 基础布局 base.njk 定义 <html> 骨架、#portal-arrival 内联遮罩（<body> 第一个元素）、导航、页脚、背景 canvas、虫洞 canvas、JS 引用

- 数据内联：base.njk 中根据 worldId 条件调用 worldDataScript filter，输出 <script>window.__WORLD_DATA__ = {...};</script>

## 5. JS 模块化细节

- 源码使用 ES modules（import/export）
- esbuild 打包为 IIFE 格式 dist/js/bundle.js（62KB），virtual-timeline.js 单独复制
- BackgroundManager 从 hardcode getElementById 改为 init(canvasId, videoId, imageId) 模式

- 到达侧：initPortalArrival 检测 location.hash → 临时关闭虚拟滚动 → 全量渲染 → 定位目标 → 反向收束动画 → dismissPortal → 恢复虚拟滚动

## 7. 传送门过渡（Portal Transition）

出发侧：
  1. 用户点击 .cross-world-link（e.preventDefault）
  2. 以点击坐标为圆心，Canvas 涡旋粒子向外螺旋扩散
  3. 光晕完全不透明时 → window.location.href = targetUrl

到达侧：
  1. #portal-arrival 内联 div 从 HTML 解析第一刻遮住全屏（#060a14 深空底色）
  2. 读取 window.__WORLD_DATA__（内联数据，瞬时可用）
  3. 解析 location.hash 得到目标事件 ID
  4. 临时关闭虚拟滚动 → 全量渲染所有节点
  5. scrollTo + requestAnimationFrame 等待目标节点
  6. 播放反向收束动画（Canvas 粒子 + 光晕收缩到目标位置，~400ms）
  7. 移除遮罩，恢复虚拟滚动
