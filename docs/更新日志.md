## V2.5.4 / #d4a060

移动端返回键关闭卡片 · 滚动条暗金皮肤

- **返回键关闭卡片**：事件模态框集成 History API，打开时推入历史记录，按浏览器返回键关闭卡片而非跳回星图首页。✕ 按钮、遮罩点击、Esc 键统一由 popstate 驱动关闭，历史栈无残留。连续点不同卡片不重复推记录。
- **滚动条暗金皮肤**：事件卡片滚动条改为暗底金条样式（WebKit 伪元素 + Firefox scrollbar-color），与网站暗金主题协调。

## V2.5.3 / #c0a860

数据 JSON 体积优化 — 去冗余 · 紧凑输出

- **JSON 去冗余**：`validate-data.js` 的 `flattenBranchEvents()` 原先同时输出 `eras`（事件嵌套）和 `events`（完全重复的扁平副本），导致 JSON 体积翻倍。现移除冗余的 `events[]` 数组，仅输出 `eras`。`data-loader.js` 运行时从 `eras` 重建 `events` 供 `findEventById` O(1) 查找。
- **紧凑 JSON 输出**：`JSON.stringify(data, null, 2)` → `JSON.stringify(data)`，去除缩进空白。
- **效果**：ff14.json 2.5MB→1.2MB (-52%)，arknights.json 1.7MB→557KB (-67%)，总数据 JSON 4.6MB→2.0MB (-58%)。

## V2.5.2 / #8ab898

生产构建优化 — JS压缩 · defer加载 · 源映射清理 · 图片瘦身

- **esbuild 生产优化**：`minify: true` + `sourcemap: false` + `drop: ['console']`。bundle.js 1.2MB→551KB，star-map-3d.js 1MB→512KB。
- **Script defer**：所有 script 标签加 `defer`，不阻塞 HTML 解析。
- **源映射清理**：构建后自动删除旧 .map 文件，CI 验证无残留。
- **图片瘦身**：已优化为 WebP 的原图自动从 dist 清理。

## V2.5.1 / #d4a060

滚动性能修复 · 重复处理器清理 · 死代码移除

- **修复重复滚动处理器**：`virtual-timeline.js` 的 scroll handler 与 `timeline-ui.js` 完全重复且无节流，每帧触发两次 `VirtualTimeline.update()` + `updateEraNavHighlight()`（各遍历全部 876 items）。现移除 virtual-timeline.js 的 scroll 监听器，仅保留 timeline-ui.js 的 RAF 节流版本。resize handler 维持不变。
- **清除 `window.renderEvents` 死代码**：该全局函数内部调用仅在 ES module 中存在的 `getBranchEvents`，导致 `ReferenceError`。V2.5.0 已移除唯一调用点，现完全删除。`renderEventsDirect` 保留为 `VirtualTimeline._renderEventsDirect` 备未来使用。

## V2.5.0 / #7eb8a0

渲染管线重构 · 传送门性能修复 · 图片优化管线 · 容错与缓存

- **传送门到达侧废除全量渲染**：`portal-transition.js` 重写 `initPortalArrival()`——原逻辑"关闭虚拟滚动 → 全量 DOM 构建 → 滚动 → 还原虚拟滚动"在 876 事件下引发 DOM 爆炸和主线程阻塞。现改为直接 `VirtualTimeline.load()` + `estimateScrollTopByEventId()` 预估定位，仅渲染视口 ~15 个节点，动画完成后无需二次加载
- **虚拟滚动引擎扩展**：`VirtualTimeline` 新增 `estimateScrollTopByEventId(eventId)` 纯函数，遍历已加载 item 列表累加高度返回目标偏移；到达侧若目标在首屏 20 个事件内则跳过手动滚动
- **网络异常容错**：`data-loader.js` 新增 `fetchWithRetry()`（首请求失败后等待 1s 重试 1 次）+ `DataLoadError` 自定义错误类；失败后 UI 层渲染"星图信号中断"主题错误卡片含重连按钮，替代原先 cover 文字替换和白屏
- **图片优化管线**：接入 `@11ty/eleventy-img`，`build.js` 新增 `buildImages()` 步骤——扫描数据 JSON 中引用的图片 → 生成 320/640/960w WebP → 后处理 `dist/data/*.json` 自动将 `src` 指向 960w WebP。FF14 162 张引用图片 7.6MB → 1.3MB（83% 缩减）
- **渐进式 JSON Schema 扩展**：`validate-data.js` 新增 `images[].width` / `images[].height` 可选字段校验（若存在则必须是 Number，不存在不报错）
- **图片渲染优化**：`timeline-ui.js` 首张图片 `loading="eager"`、余图 `lazy`；全部 `decoding="async"` 异步解码；数据提供 width/height 时直接设置属性消除 CLS
- **Fetch 缓存控制**：`.eleventy.js` 注入 `buildVersion`（CI 用 GITHUB_SHA，本地用时间戳 hash），`base.njk` 写入 `<body data-version>`，`data-loader.js` 模块初始化时读取并追加 `?v=` 到所有 fetch URL，发版后自动打破缓存
- **JSDoc 类型注解**：`data-access.js` 新增 `@typedef`（World / Event / Branch / ImageData / CrossRef），`data-loader.js` 新增 `@typedef Era`，三个核心文件全部 export 函数补齐 `@param`/`@returns`
